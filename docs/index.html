HomeNav Agent

一个基于 Habitat 的中文具身导航 Agent。用户可以输入 请到沙发旁边、请到桌子旁边、请到植物旁边 等指令，Agent 会基于 RGB-D 视觉、机器人位姿、地图构建和路径规划，导航到目标物体旁边。 任务完成后，Agent 会回复：任务完成。还需要什么？

Habitat 仿真环境 RGB-D 目标导航 中文文字输入 可达 standoff 规划 多目标导航

项目目标

本项目选择“导航方向”。系统在 Habitat 居家场景中运行，只使用仿真器提供的视觉观测 和机器人本体位姿，不使用语义地图或真实目标坐标等特权信息。Agent 可以根据用户输入的 目标物体，构建可通行地图，选择目标附近可达的安全停靠位置，并执行导航。

系统流程

感知与目标记忆

1. 解析中文指令，得到目标物体类别。
2. 使用 RGB 图像进行目标检测。
3. 多帧确认目标，避免单帧误检。
4. 对多个同类物体，选择更近或更容易到达的实例。

建图与规划

1. 使用 depth 观测构建 top-down 可通行地图。
2. 对障碍物进行膨胀，保证机器人安全距离。
3. 将检测到的物体作为 semantic anchor。
4. 在目标附近寻找 reachable standoff cell 并执行路径跟随。

额外目标

关键设计

物体不是导航终点，而是语义锚点

系统不会直接追 bounding box 中心。检测到的物体只作为目标锚点， 机器人真正要到达的是目标附近可通行、可到达的 free cell。

Reachable Standoff Planning

用户说“到沙发旁边”并不意味着撞向沙发中心。系统会在可达区域中搜索目标附近的 standoff cell，从而停在物体旁边而不是物体内部。

多实例目标选择

当环境中出现多个同类目标时，系统不会死守最早看到的实例，而是选择当前地图上 更容易到达的 confirmed instance，类似人类会选择更近、更方便的目标。

持续学习接口

项目额外提供了持续学习数据接口，可以将导航日志导出为 intervention-ready episodes。 该接口预留了 human_action 和 intervened 字段，未来可以进一步接入 LeRobot-style 数据集，用于模仿学习或强化学习。

Bonus：持续学习数据接口

本项目没有声称已经完成完整的 RL 训练，而是实现了一个持续学习数据接口。 脚本 scripts/export_intervention_dataset.py 可以把导航过程中的 run log 转换成 episode-level 数据。

导出结果包括 learning_data/intervention_episodes.jsonl 和 learning_data/summary.json。其中每一步包含 observation 路径、 planner action、executed action、success label 和预留的人类干预字段。
