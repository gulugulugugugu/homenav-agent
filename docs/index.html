HomeNav Agent

A Habitat-based embodied navigation agent that accepts natural-language commands such as 请到沙发旁边, builds an RGB-D traversibility map, and navigates to a reachable standoff position near the requested object.

Habitat Simulation RGB-D ObjectNav Natural Language Input Reachable Standoff Planning Multiple Target Objects

Project Goal

This project chooses the navigation track. The embodied agent runs inside a home-like Habitat scene and uses only simulator visual observations plus robot pose. It does not use privileged semantic maps or ground-truth object coordinates. After completing a navigation command, the agent replies: 任务完成。还需要什么？

System Pipeline

Perception and Memory

1. Parse user text into a target object.
2. Run RGB object detection for table, sofa, plant, TV, etc.
3. Confirm detections across frames and store object instances.
4. For multiple same-class objects, select the easiest reachable instance.

Mapping and Planning

1. Convert depth observations into a top-down free/obstacle map.
2. Inflate obstacles for robot safety.
3. Search reachable cells near the semantic object anchor.
4. Follow a map-based path to a standoff cell near the target.

Key Design Choices

Object as Semantic Anchor

The agent does not simply chase a bounding box center. A detected object is treated as a semantic anchor, while the robot plans to a nearby reachable free cell.

Reachable Standoff Goal

The robot searches the reachable component of the map and selects a safe standoff location near the target object, which avoids driving into furniture.

Multi-Instance Selection

When there are multiple objects of the same class, the system can switch to the nearest or easiest reachable confirmed instance, similar to human navigation behavior.

Continual Learning Extension

The system records observations, detections, planned paths, actions, and success/failure states. These logs can later be extended into human-intervention episodes and converted into a LeRobot-style dataset for continual learning.
